## 1. NGINX高危堆溢出漏洞PoC代码已公开
- 分类: 漏洞与补丁 / Vulnerabilities & Patching
- 来源: SecurityWeek
- 发布时间: 2026-05-16T10:02:00+00:00
- 风险等级: 高
- 关键词: NGINX漏洞, CVE-2026-42945, 堆缓冲区溢出, PoC公开, 远程代码执行
- 命中关注词: cisco
- 原文链接: https://www.securityweek.com/poc-code-published-for-critical-nginx-vulnerability/
- 摘要来源: 大模型

- 原标题: PoC Code Published for Critical NGINX Vulnerability

### 摘要
NGINX曝出严重堆缓冲区溢出漏洞（CVE-2026-42945，CVSS 9.2），相关PoC利用代码已公开。该漏洞存在于重写模块中，攻击者可通过构造特定URI触发拒绝服务，若服务器禁用ASLR则可能实现远程代码执行。F5已发布补丁修复此存在16年的缺陷，建议管理员立即升级并启用内存保护机制。

### 要点
- 漏洞CVSS评分高达9.2且PoC已公开，攻击者可利用特定URI触发拒绝服务或远程代码执行。
- 漏洞根源在于重写模块双遍处理机制导致缓冲区分配过小，禁用ASLR将显著增加RCE风险。
- 建议NGINX用户立即应用F5最新补丁，并确保服务器全面启用ASLR等内存随机化防护。

## 2. 多Agent系统面临“授权传播”结构性风险
- 分类: AI安全与安全智能体 / AI Security & Security Agents
- 来源: 安全内参
- 发布时间: 2026-05-16
- 风险等级: 中
- 关键词: 多Agent系统, 授权传播, AI安全, 身份治理, 权限越界
- 命中关注词: 无
- 原文链接: https://www.secrss.com/articles/90379
- 摘要来源: 大模型

- 原标题: 提示注入之外：多Agent系统正在暴露“授权传播”风险

### 摘要
研究指出多智能体系统除提示注入外，正面临更底层的“授权传播”结构性风险。在复杂工作流中，权限若未随数据检索、任务拆分与结果合成持续校验，极易导致越权访问或敏感信息泄露。论文强调身份治理应作为底层基础设施设计，而非依赖单次网关检查。

### 要点
- 多Agent工作流中权限传递缺乏持续校验，即使无提示注入也可能引发越权与数据组合泄露。
- 身份治理需从中间件升级为底层基础设施，贯穿任务委托与数据合成的全生命周期。
- 建议企业在部署AI Agent时建立动态权限追踪与最小化授权机制，确保执行链可审计。
