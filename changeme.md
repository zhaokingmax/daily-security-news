# ChangeMe

## 0.9
- 写报告前会对已存在的旧条目自动补中文标题，避免旧 JSON 合并后仍长期保留英文标题。
- 下次重新生成当天日报时，旧数据也会跟随升级成中文标题优先、顶部中英双语显示。

## 0.8
- 大模型输出结构新增 `title_zh`，要求同时生成中文标题与中文摘要。
- 顶部分类清单改为中英双语标题显示：优先中文标题，后接原始英文标题。
- 详情区域标题改为优先显示中文标题，并在需要时单独保留原始标题。
- 分类标题改为中英双语展示，提升快速浏览体验。
- 回退摘要场景新增标题翻译补救，尽量减少“摘要中文、标题英文”的情况。

## 0.7
- 新增分阶段进度日志：按数据源抓取、候选累计、正文抓取进度、LLM 批次发送/返回、重试情况都会实时输出。
- 正文抓取阶段会显示并发进度，便于判断是卡在抓源、抓正文还是卡在大模型。

## 0.6
- GitHub Actions 改为 `python -u -m src.main` 并启用 `PYTHONUNBUFFERED=1`，确保日志实时输出。
- 入口阶段新增启动、加载状态、抓取、摘要、写文件等进度日志，便于定位卡住的阶段。
- Python 运行时标准输出和错误输出改为行缓冲，减少 GitHub Actions 长时间静默。

## 0.5
- 先收集候选链接再抓正文，只为最终高优先级候选抓取全文，显著降低 GitHub Actions 总耗时。
- 正文抓取改为并发执行，减少单日 40-50 条资讯时的等待时间。
- 默认批量摘要从 `5` 提升到 `8`，单篇输入从 `2500` 缩短到 `1800` 字，进一步减少 token 消耗。
- GitHub Actions 升级到 `actions/checkout@v5` 和 `actions/setup-python@v6`，修复 Node.js 20 弃用警告。
- 工作流超时时间提升到 45 分钟，并补充批处理进度日志，方便定位卡住的位置。

## 0.4
- 黑名单改为只基于标题和摘要判断，不再扫描整篇正文，避免因正文顺带提到 `CVE` 等词导致误杀。
- 黑名单日志会显示命中的关键词，便于排查为什么被过滤。

## 0.3
- 大模型摘要改为批量调用，失败文章自动单篇重试，尽量减少回退摘要出现的次数。
- 新增 `LLM_BATCH_SIZE` 和 `MAX_LLM_INPUT_CHARS_PER_ARTICLE` 配置，用于降低 token 消耗。
- 新增 `LLM_RETRY_COUNT`，遇到模型偶发失败会自动重试。
- 日报顶部新增分类清单，按主流安全主题聚合标题，支持快速浏览。
- 为每条资讯增加分类字段，覆盖 AI 安全、APT、运营商、EDR/XDR、威胁情报等 10+ 个主题。
- 默认抓取规模提升到每日 50 条、单源 8 条，关键词优先，不够时自动补充非关键词资讯。
- 工作流超时时间提升到 30 分钟，适配更高的每日资讯量。

## 0.2
- 数据输出改为按月归档，日报保存到 `data/YYYY-MM/YYYY-MM-DD.{json,md}`。
- 新增关注关键词优先机制，支持中英文关键词匹配并在报告中标记命中词。
- 新增黑名单过滤机制，默认过滤 `密码学`、`cryptography`、`CVE` 相关内容。
- 增强大模型提示词，要求英文资讯先翻译再生成中文摘要，并在必要时做二次中文化修正。
- 扩充资讯源，新增 CSO Online、Infosecurity Magazine、CyberScoop、The CyberWire、安全内参、安全客、CNCERT。

## 0.1
- 初始版本：按天抓取网络安全 RSS，生成日报并写回仓库。
